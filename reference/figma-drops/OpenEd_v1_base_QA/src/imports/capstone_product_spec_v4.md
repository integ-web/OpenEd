# Capstone Product Spec V4 — Aster-3 Pre-Deployment Evaluation Dossier

## Goal
Make the capstone the strongest part of the course.

The learner must answer:
Is Aster-3 ready for restricted real-world release, and what evidence supports that recommendation?

## Product feel
- Professional evaluation review.
- Serious but not intimidating.
- Evidence-driven.
- Staged and interactive.
- Portfolio-worthy.

## Release options
The learner must recommend one:
1. Broad release.
2. Restricted release.
3. Trusted-access release only.
4. Delay pending mitigations.
5. Do not release in current form.

## Capstone stages
0. Briefing room.
1. System and release context.
2. Threat model assembly.
3. Evaluation portfolio design.
4. Evidence room.
5. Risk dashboard.
6. Threshold memo.
7. Executive report.
8. Defense simulation.
9. Portfolio export.

- CapstoneBriefingRoom

- EvaluationCardBuilder

- ThresholdMemoBuilder

## Core interaction pattern
Each stage should have:
- Scenario context.
- What the learner must decide.
- Inputs imported from course artifacts.
- Builder or decision activity.
- Quality gate feedback.
- Save state.
- Next-stage CTA.
