## Golden lesson to build perfectly

Use this as the design-quality benchmark for the whole course:

Module 3:
Benchmark Design

Lesson:
From Risk Question to Evaluation Objective

Duration:
90 minutes

Learning objective:
The learner should be able to convert a vague AI safety concern into a precise evaluation objective that can guide benchmark design, red-team tasks, evidence collection, and governance decisions.

Lesson copy:

Title:
From Risk Question to Evaluation Objective

Subtitle:
Good frontier evaluations start with the decision you need to make, not the benchmark you happen to have.

Why this matters:
Many beginners ask, “Which benchmark should we run?” Professional evaluators ask, “What risk claim are we trying to test, and what decision will the evidence inform?” A benchmark is only useful if it connects to a real threat model, a capability hypothesis, and a deployment decision.

Mental model:
Threat Model → Evaluation Objective → Task Design → Evidence → Decision

Concept cards:

* Threat model
* Capability hypothesis
* Evaluation objective
* Metric
* Evidence
* Threshold
* Mitigation

Worked example:

Weak version:
“Test whether the model is dangerous.”

Improved version:
“Evaluate whether the model, when given browser and terminal access, can meaningfully improve a novice user’s ability to complete multi-step cyber reconnaissance tasks in a controlled environment.”

Explain why the improved version is stronger:

* Names the model context
* Names the user
* Names the capability
* Names the task family
* Names the environment
* Makes measurement possible

Interactive exercise:
Give the learner this vague concern:
“This model might help with biological misuse.”

The learner fills:

* Actor
* Capability
* Harm pathway
* Model access level
* Task environment
* Success condition
* Evidence needed
* Disclosure sensitivity

Artifact output:
Evaluation Objective Card

Fields:

* Risk concern
* Threat model
* Capability hypothesis
* Evaluation objective
* Evidence needed
* Decision informed
* Confidence level

Reflection:
Think of one AI risk you have heard people discuss. Rewrite it as an evaluation objective. What decision would the evaluation inform?

Design this lesson as the polished source of truth for every other lesson.

2. Third-party evaluator access negotiation
   A frontier lab offers only four days of black-box access. The evaluator must decide whether the access supports the safety claim.

3. Biology benchmark triage
   A policy team asks for a “bio benchmark.” The draft includes harmless textbook questions, risky operational prompts, and possibly contaminated examples.
